QUEUE STATISTICS BREAKDOWN
─────────────────────────────────────────────────────────────────────────────────────────────

Status Bar Format: Q: Scan:45 Y:12 H:3 C:1
                      │     │   │   │  └─ Critical priority: 1 process
                      │     │   │   └──── High priority: 3 processes
                      │     │   └──────── YARA queue: 12 pending scans
                      │     └──────────── Scan queue: 45 monitored processes
                      └────────────────── Queue indicator

Priority Scanning Intervals:
  Critical: Every 5 seconds   (Most aggressive)
  High:     Every 12 seconds  (Frequent)
  Medium:   Every 30 seconds  (Regular)
  Low:      Every 60 seconds  (Background)


WORKFLOW EXAMPLE
─────────────────────────────────────────────────────────────────────────────────────────────

1. Analyst watches status bar:
   Q: Scan:20 Y:3 H:0 C:0  ──► Normal operations

2. Critical alert appears:
   [12:34:56] ProcessHollowing detected...  ◄── Bright RED color

3. Status bar updates:
   Q: Scan:21 Y:8 H:1 C:1  ──► Shows elevated priority

4. Analyst right-clicks alert:
   → "Browse Memory Regions"

5. Memory Regions window opens:
   Multiple RED rows visible  ──► Suspicious memory

6. Analyst selects RED row:
   → "Dump Selected" → Saves to disk

7. Offline analysis:
   → Hex editor / YARA / Disassembler


BENEFITS AT A GLANCE
─────────────────────────────────────────────────────────────────────────────────────────────

✓ Queue visibility      → Understand system load and activity
✓ Color-coded alerts    → Instant threat severity identification
✓ Visual memory regions → Spot injection/hollowing immediately
✓ No training required  → Intuitive color system (red=bad, green=good)
✓ Faster response       → Reduce mean-time-to-detect (MTTD)
✓ Better workflow       → Prioritize based on visual cues
✓ Minimal overhead      → <1% CPU, updates once per second
